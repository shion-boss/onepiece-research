## 1. 大原則 (1-3)

- **カードテキスト > 総合ルール**: 矛盾時はカードテキスト優先 (1-3-1)
- **不可能な行動は実行しない**: 効果が「N 枚引け」と命じても、デッキが N 枚なければ可能な分だけ (1-3-2)
- **既にその状態なら何もしない**: アクティブ→アクティブにしろ、は no-op (1-3-2-1)
- **禁止 > 行動**: 行動指示と禁止が同時にあれば常に禁止が優先 (1-3-3)
- **同時選択はターンプレイヤーが先**: 双方が同時に選択を求められたら、ターン側 → 非ターン側の順 (1-3-4, 1-3-10)
- **数値の下限**: 特に指定がなければ整数 ≥ 0 を選ぶ。「N まで」は 0 を選択可 (1-3-5)
- **パワーは負数になりうる**: 他の値はマイナスになっても 0 として扱う (1-3-6)

## 2. 領域 (3)

| 領域 | 公開 | 内容 |
|---|---|---|
| デッキ | 非公開 (枚数のみ) | 50枚。順序変更不可 |
| ドン!!デッキ | 公開 (枚数・順序自由) | 10枚 |
| 手札 | 自分のみ閲覧可 | **手札上限なし** |
| トラッシュ | 公開 | KO/発動済イベント等 |
| リーダーエリア | 公開 | リーダー1枚。ここから移動しない (3-6-3) |
| キャラクターエリア | 公開 | **最大5枚** (3-7-6) |
| ステージエリア | 公開 | **最大1枚** (3-8-5) |
| コストエリア | 公開 | ドン!!カードの置き場 |
| ライフ | 非公開 (枚数のみ) | 裏向き |

> **「場」 = リーダー / キャラ / ステージ / コストエリア の総称** (3-1-2)
>
> エリア間移動時、**前エリアで適用されていた効果は失われる** (3-1-6)。新領域では新しいカード扱い。これにより `static_buff` 等は領域移動時にリセットすべき。
>
> エリア超過時の差替: キャラ 5 枚 / ステージ 1 枚状態で新規登場させる場合、既存 1 枚を **持ち主のトラッシュへ** (3-7-6-1, 3-8-5-1)。これは **ルール処理であり、効果ではない** ので「KO 時」「離れたとき」等のトリガーは発動しない (3-7-6-1-1)。

## 3. ゲーム開始 (5)

1. デッキシャッフル → 所定の場所に置く
2. リーダー表向きで配置
3. ジャンケン → 勝者が先攻/後攻を選ぶ
4. デッキから 5 枚ドロー
5. **マリガン**: 先攻から順に各プレイヤーは 1 度だけ、手札全部をデッキに戻して 5 枚引き直し可
6. 自分のリーダーのライフ枚数だけ、デッキ上から裏向きでライフへ (デッキ最上だったカードがライフ最下になるよう順に)
7. 先攻からゲーム開始

## 4. ターン進行 (6)

```
リフレッシュ → ドロー → ドン → メイン → エンド
```

### リフレッシュ (6-2)

1. 「ターン開始時まで」の継続効果が無効化
2. 「ターン開始時」の効果発動
3. 自分のリーダー/キャラに付与されているドン!!すべてを **レストでコストエリアへ戻す**
4. 自分の場のレストカードすべてをアクティブに

> 注: engine は現在「アクティブで戻す」になっている可能性あり。仕様は **レスト**。実装確認のこと。

### ドロー (6-3)

- 1 枚引く
- **先攻 1 ターン目はドロー不可**

### ドン (6-4)

- ドン!!デッキから **2 枚** をアクティブでコストエリアへ
- **先攻 1 ターン目のみ 1 枚**
- ドン!!デッキが 1 枚しか無ければ 1 枚、0 枚なら何もしない

### メイン (6-5)

以下を **好きな順番で何度でも**:
- カードのプレイ (キャラ登場 / ステージ登場 / 【メイン】イベント発動)
- 効果の発動 (【起動メイン】等)
- ドン!!の付与
- バトル

「メインフェイズ終了」を宣言するとエンドフェイズへ。

### エンド (6-6)

順番:
1. **【自分のターン終了時】効果** をターンプレイヤーが任意順で全解決
2. **【相手のターン終了時】効果** を非ターンプレイヤーが任意順で全解決
3. ターンプレイヤーの継続効果のうち「このターン終了時/エンド終了時まで」を無効化
4. 非ターンプレイヤーの同上
5. ターンプレイヤーの「このターン中」期限を無効化
6. 非ターンプレイヤーの「このターン中」期限を無効化
7. ターン交代 → 次のターンのリフレッシュへ

> engine 実装ヒント: `turn_buff` のリセットは Phase.END で **両プレイヤー両方の InPlay** に行う。ターンプレイヤー側を先、非ターン側を後で行うのが厳密だが、結果に差はほぼ出ない。

## 5. カードのプレイ (4-7, 6-5-3, 2-7)

- **キャラ登場**: 公開 → コスト枚数のドンをレスト → アクティブで CharacterArea へ
- **ステージ登場**: 同上 (StageArea へ。既存があればトラッシュへ)
- **イベント発動**: 公開 → コスト払う → トラッシュへ → 効果発動
- 登場したキャラは **そのターン中アタック不可** (3-7-4) ※【速攻】例外

## 6. ドン!!の付与 (6-5-5)

- アクティブのドン!!1 枚をリーダー or キャラの下に重ねる
- 自分のターン中、ドン!!1 枚につき **+1000 power**
- 何度でも付与可
- **ドン付与カードが領域移動 (KO等) するとき、ドン!!はレストでコストエリアへ戻る** (6-5-5-4)

## 7. バトル (7)

```
アタック宣言 → ブロック → カウンター → ダメージ → バトル終了
```

### アタックステップ (7-1-1)

1. **両プレイヤーの 1 ターン目はバトル不可** (6-5-6-1)
2. アクティブのリーダー/キャラ 1 枚をレスト → アタック宣言
3. 対象選択: 相手リーダー or **レストの相手キャラ 1 枚**
4. 【アタック時】【相手のアタック時】等の効果発動

### ブロックステップ (7-1-2)

- 受け側は自分の場の【ブロッカー】カードのうち **アクティブのもの 1 枚をレストにして** ブロックを発動可能 (10-1-4)
- 「アタックされたカードを【ブロッカー】を発動したカードに変更」する
- 【ブロック時】効果が発動

### カウンターステップ (7-1-3)

受け側は以下を好きな順で何度でも実行可:
- **(アイコン)カウンター**: 手札の (アイコン)カウンター 持ちキャラをトラッシュへ → 自リーダー/キャラ 1 枚のパワーを **そのバトル中** カウンター値分上昇
- **【カウンター】イベント**: 手札のイベントのコストを払ってトラッシュへ → 効果発動

### ダメージステップ (7-1-4)

- アタッカーパワー **≥** 防御カードパワー → 勝ち
  - リーダー対象: **1 ダメージ** → 相手はライフ上 1 枚を手札へ (or トリガー発動)
    - ダメージ確定時に相手ライフ 0 → アタッカー側勝利
  - キャラ対象: そのキャラを **KO** (トラッシュへ)
- アタッカーパワー **<** 防御パワー → アタッカー負け、何も起きない

### バトル終了 (7-1-5)

- 「バトル終了時」効果発動
- 「このバトル中」期限の効果が無効化

### トリガー (7-1-4-1-1-2, 10-1-5)

- リーダーがダメージを受ける → ライフ上カード 1 枚を **自分だけ確認**
- そのカードに【トリガー】があれば **公開して効果発動 (代わりに手札に加えない)** を選べる
- 発動を選ばない場合、公開せず手札に加える
- 【トリガー】発動中、そのカードはどの領域にも属さない (10-1-5-3)。発動完了後、特に指示が無ければ持ち主のトラッシュへ
- 【ダブルアタック】等で 2 ダメージ以上の場合、1 ダメージずつ処理を繰り返す (4-6-2-2, 7-1-4-1-1-3)

## 8. 効果の分類 (8-1)

| 種類 | 説明 | 例 |
|---|---|---|
| **自動効果** | 事象発生時に自動発動 | 【登場時】【アタック時】【ブロック時】【KO 時】【自分/相手のターン終了時】、「〜した時」「〜された時」 |
| **起動効果** | ターンプレイヤーがメインフェイズ中に発動宣言 | 【起動メイン】【メイン】 |
| **永続効果** | 有効期間中、常に発生し続ける | 「〜の場合」「自分のターン中」常在条件など |
| **置換効果** | 「代わりに」と表記 | 【トリガー】【バニッシュ】 |

> **重要**: engine の `on_attached_don` は永続効果カテゴリ (8-1-3-3)。ドン枚数が条件を満たす間、常に効果を発生し続ける。条件が外れると効果が解除される。

## 9. 発動コストと条件 (8-3)

- `①` `②` `③` `④`: コストエリアのアクティブのドン!! N 枚をレスト = コスト
- `ドン!!-N`: 自分の場 (リーダー/キャラ/コストエリア) のドン!! N 枚をドン!!デッキに戻す = コスト
- 効果コスト 「:」 効果本体 (例: 「【起動メイン】②: 1ドロー」)
- 「することができる」「にできる」 = 任意の発動コスト (払わない選択可、その場合効果発動なし)

## 10. キーワード効果 (10-1)

| キーワード | 意味 |
|---|---|
| 【速攻】 | 登場ターン中もアタック可能 |
| 【速攻：キャラ】 | 登場ターン中も **相手キャラへのみ** アタック可能 |
| 【ダブルアタック】 | アタックでリーダーライフに与えるダメージが 1 → 2 に置換 |
| 【バニッシュ】 | アタックでリーダーライフにダメージを与えた場合、ライフは手札へ加わらず **トラッシュへ**。**【トリガー】は発動しない** |
| 【ブロッカー】 | ブロックステップで、自カード以外がアタックされた時、自カード(アクティブ)をレストにしてアタック対象を自カードへ変更 |
| 【トリガー】 | ライフから手札に加える代わりに公開して効果発動 (置換効果) |
| 【ブロック不可】 | このカードのアタック中、相手は【ブロッカー】を発動できない |

## 11. キーワード (10-2)

| キーワード | 意味 |
|---|---|
| KO | バトル敗北 or 「KO する」効果でキャラがトラッシュへ。それ以外でトラッシュへ送られた場合は KO ではない |
| 【KO 時】 | KO される時。自動効果のうち特例で、効果解決前にキャラはトラッシュへ移動済の状態で発動 (10-2-17-2) |
| 【起動メイン】 | バトル以外のメインフェイズに発動可 |
| 【メイン】 | イベントカード専用。バトル以外のメインフェイズで発動可 |
| 【カウンター】 | イベントカード専用。相手ターンのカウンターステップで発動可 |
| 【アタック時】 | 自分のアタック宣言時 |
| 【相手のアタック時】 | 相手のアタック宣言時、相手の【アタック時】の後 |
| 【自分/相手のターン終了時】 | エンドフェイズ |
| 【ドン!!×N】 | このカードに N 枚以上のドン!! 付与で条件満たす (永続条件) |
| 【自分/相手のターン中】 | そのターン中であること |
| 【ターン1回】 | そのターン中 1 度だけ発動と解決可。同名カードが複数枚あっても各 1 回。**領域移動して場に戻った場合は別カード扱いなので再発動可** (10-2-13-4) |
| 捨てる | 手札からトラッシュへ |

## 12. 効果の解決順 (1-3-4, 1-3-10, 8-6, 6-6-1-1)

複数効果が **同時に** 発動タイミングを満たした場合:

1. **ターンプレイヤー側の効果から先**、任意の順で全解決
2. その後、非ターンプレイヤー側を任意の順で全解決

ターンプレイヤー解決中に新しい効果のタイミングが満たされた場合、そのまま続行。すべてのターンプレイヤー側を解決後に非ターン側へ移る。

## 13. ルール処理 (9, 1-2)

ルール処理は他の行動の実行中であっても **発生時点で即座に解決**。

### 敗北判定処理 (9-2-1)

ルール処理開始時に、以下を満たすプレイヤーは敗北:
- ライフ 0 枚 + リーダーがダメージを受けた状態
- デッキが 0 枚

複数プレイヤーが同時に満たすこともあり得る (引き分け)。

## 14. デッキ構築 (5-1-2)

- リーダー 1 枚ちょうど
- メインデッキ 50 枚ちょうど (キャラ + イベント + ステージ)
- ドン!!デッキ 10 枚ちょうど
- リーダーの色を **含む色** のカードのみ採用可
- **同一カードナンバー** は 4 枚以下 (= base_id 単位。再録/パラレル違いも合算)

## 15. その他重要な仕様

### 複数色 (2-3-4〜2-3-6, 1-3-1)
- 持つすべての色のカードとして扱う (赤緑カードは赤かつ緑)
- 「多色」 = 複数色を持つカード

### 元々の (4-9)
- 「元々のパワー」「元々のコスト」 = カードに表記された原始値
- 永続効果で変更されていても元々の値は変わらない

### 「場合」と「その後」 (4-10)
- 「場合」前文が解決不能 → 後文も解決しない
- 「その後」前文が解決不能 → 後文は解決する

### 離れる (4-11)
- ある領域から別の領域へ移動すること

### パワー0 にする (4-12)
- 効果発動時の対象パワーと同じ数値分マイナスする継続効果

### 永久循環 (11-1)
- 双方とも止められない無限ループ → 引き分け
- 一方のみ止められる → そのプレイヤーが回数を宣言して停止
- 両方止められる → ターン側 → 非ターン側の順に回数宣言、小さい方の回数で停止

## 17. engine とルールの差分 (修正履歴 + 残課題)

## 18. 公式 Q&A データの参照方法 (`db/faq/`)

`db/faq/` 以下に公式 FAQ + 弾別 cardqa が JSON として保存されている。**ルール裁定や個別カードの挙動について「これって発動する? しない?」みたいな質問が来たら、まず grep で当たる**。

### ファイル構成

| ファイル | 件数 | 内容 |
|---|---|---|
| `db/faq/base.json` | 46 | 基本ルール (ライフ・登場・コスト・トリガー等) |
| `db/faq/keyword_effect.json` | 38 | キーワード効果 (登場時・KO 時・トリガー等) |
| `db/faq/keyword.json` | 26 | キーワード (ブロッカー・速攻・ダブルアタック・バニッシュ等) |
| `db/faq/detail.json` | 30 | 詳細ルール (置換効果・優先順・解決順等) |
| `db/faq/cardqa_<series>.json` | 2,500+ | 弾別 (op_01〜op_15, st_01〜st_30, eb_01〜eb_04, prb_01/02 等) のカード個別 Q&A |
| `db/faq/INDEX.md` | - | サマリ (件数 + 一覧) |

### grep の例

```bash
# 全 FAQ から「トリガー」を含む Q&A を探す
jq -r '.items[] | select(.q | contains("トリガー") or .a | contains("トリガー")) | "Q: " + .q + "\nA: " + .a + "\n"' db/faq/*.json

# 特定カード (例: OP07-115) の cardqa を探す
grep -l "OP07-115\|助けてクエーサ" db/faq/cardqa_*.json
jq '.items[] | select(.q | contains("OP07-115"))' db/faq/cardqa_op_07.json

# キーワード「ダブルアタック」
jq '.items[] | select(.q + .a | test("ダブルアタック"))' db/faq/*.json
```

### 各エントリの形式

## 19. 禁止/制限/禁止ペア (`db/banlist/master.json`)

### 現行 (`master.json` 取得時点)

| 区分 | 件数 |
|---|---|
| 禁止カード | デッキに 1 枚も入れられない |
| 制限カード | デッキに 1 枚のみ |
| 禁止ペア | A が入っているデッキには B を組み合わせ不可 |

### deck builder への組み込み (推奨)

- `engine/deck.py` の `validate()` で禁止カード/制限カード違反を検知すべき
- 現状未実装。`db/banlist/master.json` を読み込んで違反を返すロジックを追加するのが次の改善

### 取得元

`https://www.onepiece-cardgame.com/rules/restriction/` (302 リダイレクト先のマスター頁)。`scripts/scrape_official_banlist.py` 実行で `db/banlist/master.json` を再生成。

### deck validate との統合

`engine/deck.py` の `DeckList.validate()` は `db/banlist/master.json` を自動ロードして以下をチェック:
- 禁止カード採用 (1 枚でも違反)
- 制限カード超過 (2 枚以上で違反)
- 禁止ペア両搭載 (リーダーも含めて判定)

`validate(banlist={})` を渡すと banlist 検証をスキップ可能 (テスト用)。

## 19.5. 効果オーバーレイの自動抽出 (`scripts/suggest_overlay_from_cards.py`)

`db/card_effects.json` を 0 から手書きするのは膨大なので、半自動抽出スクリプトを用意した。

```bash
.venv/bin/python scripts/suggest_overlay_from_cards.py
```

動作:
1. `cards.json` の text / trigger 欄をルールベースでパース
2. 【登場時】【KO時】【トリガー】【アタック時】【起動メイン】【自分のターン終了時】を when にマップ
3. テキスト中の「N 枚を引く」「KOする」「パワー+N」「ライフを手札」「サーチ」等を primitive に変換
4. `db/card_effects.suggestions.json` に出力 (現状 800 枚以上の候補)
5. **人間が目視レビューして必要なものだけ `card_effects.json` にマージ**

抽出は簡略化判断 (例: 「コスト4以下のキャラを KO」→ `any_opponent_character_le_5000` に統一) を含むので、誤抽出は手動で修正する前提。`_text` フィールドに元の抜粋が入るのでレビューしやすい。

## 20. 実装上の質問への回答 (FAQ)

- **Q. リーダーは攻撃できるか?** → はい。リーダーカードもアクティブなら攻撃宣言可 (7-1)。**ただし両プレイヤーの 1 ターン目は不可** (6-5-6-1)
- **Q. キャラは登場ターンに攻撃できるか?** → 不可。**速攻持ちは可** (3-7-4, 10-1-1)
- **Q. アクティブのキャラには攻撃できるか?** → 不可。攻撃対象は **相手リーダー or レストの相手キャラ** (7-1-1-2)
- **Q. ブロッカーは何度使えるか?** → 1 バトル中 1 度。発動時にレストする (10-1-4-1, 7-1-2-1)
- **Q. カウンターは何度使えるか?** → カウンターステップ中、好きな数だけ重ねられる (7-1-3-1)
- **Q. ドン付与の上限は?** → ルール上限なし (パワー上限もなし)
- **Q. リーダーへのダブルアタックは何ライフ削る?** → 2 ライフ (1-1-1-1-3 等)。**キャラ攻撃時のダブルアタックは仕様未明** (条文上明示なし → カードテキスト依存)
- **Q. 効果でキャラを KO した時 KO 時トリガーは?** → 効果による KO でも【KO 時】は発動 (10-2-1-3)。**ただし戦闘以外で「トラッシュに置く」と書かれている場合は KO 扱いにならない**
- **Q. ターン 1 回効果のキャラが KO されて再登場したら?** → 別カード扱いで再度ターン 1 回が使える (10-2-13-4, 3-1-6)
